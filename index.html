Git

Commands

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • git add -A: stages the changes made to all files for a commit
  • git pull origin branch-name: syncronices local branch with remote branch in GitHub.
  • git commit -m "commit message": commits staged files to local repository
  • git push origin branch-name: pushes commit to GitHub repository

Notes

  • Git is a version control software for developers.

Git Flow

  1. Make sure you're working in a feature branch using the command git status. If not, change to that branch using the command git checkout branch-name.
  2. Use the command git add -A to stage the changes made.
  3. Use the command git commit -m "commit message" to commit the changes.
  4. Use the command git pull origin main to check current branch is up to date before opnening a pull request.
  5. Use the command git push origin branch-name to push up the commit to the GitHub repository.
  6. Navigate to the pull requests section in the repository on Github and create a new pull request with a brief description of the changes made.
  7. Merge pull request.
  8. Use the command git pull origin main to update the main branch
